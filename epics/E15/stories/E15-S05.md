# E15-S05 — Wrapper Capacitor Android: projeto Gradle + build internal track

## Objetivo

Configurar o projeto Android (Gradle) gerado pelo Capacitor (E15-S02) para produzir um `.aab` (Android App Bundle) distribuível via Google Play Internal Testing track, com o CI (GitHub Actions runner Linux/ubuntu) realizando o build automaticamente. Após esta story, qualquer pessoa convidada no Internal Testing do Google Play Console consegue instalar o app Kalibrium em um dispositivo Android real e abrir a tela de login.

Esta story depende de **conta Google Play Console ativa** (pré-requisito externo do PM). Sem a conta, o AC de upload ao Internal track fica pendente, mas o build do `.aab` local pode ser verificado independentemente.

## Escopo

- Configuração do `android/app/build.gradle`: applicationId, versionCode, versionName, targetSdkVersion (Android 14 / API 34), minSdkVersion (Android 10 / API 29)
- Keystore de release configurado via secrets do GitHub Actions (não commitado no repositório)
- Workflow GitHub Actions `.github/workflows/build-android.yml`: trigger manual + trigger em tag `v*`
  - Runner: `ubuntu-latest`
  - Steps: checkout → setup Node → setup Java 17 → npm install → npm run build → cap sync android → gradlew bundleRelease → sign `.aab` → upload Internal track via `google-play` action
- `android/fastlane/Fastfile` com lane `build_internal`
- Ícones Android (mipmap hdpi/xhdpi/xxhdpi/xxxhdpi + adaptive icon `ic_launcher_foreground.xml` + `ic_launcher_background.xml`)
- Splash screen via `@capacitor/splash-screen` com logo Kalibrium
- Permissões declaradas no `AndroidManifest.xml`: `USE_BIOMETRIC`, `USE_FINGERPRINT`, `CAMERA`, `ACCESS_FINE_LOCATION`, `INTERNET`, `ACCESS_NETWORK_STATE`
- Configuração de ProGuard/R8 para release build
- Teste de smoke: app abre no emulador Android (API 29+) e renderiza rota `/login` sem crash

## Fora de escopo

## Critérios de aceite

### AC-001 — Build debug local passa sem erro
**Dado** que o desenvolvedor tem Java 17 + Android SDK instalados
**Quando** executa `cd android && ./gradlew assembleDebug`
**Então** o comando termina com exit 0 e `android/app/build/outputs/apk/debug/app-debug.apk` existe

### AC-002 — App abre no emulador Android sem crash
**Dado** que existe um AVD rodando (Android API 29+)
**Quando** executa `npx cap run android --target <emulator-id>`
**Então** o app abre no emulador e a rota `/login` renderiza sem crash — verificado via Espresso smoke test ou inspeção visual

### AC-003 — Ícones adaptativos declarados e presentes
**Dado** que o projeto Android foi configurado
**Quando** se executa `ls android/app/src/main/res/mipmap-hdpi/ android/app/src/main/res/mipmap-xxxhdpi/`
**Então** arquivos `ic_launcher.png` e `ic_launcher_round.png` existem em hdpi, xhdpi, xxhdpi e xxxhdpi; arquivo `ic_launcher_foreground.xml` existe em `mipmap-anydpi-v26/`

### AC-004 — AndroidManifest.xml contém permissões obrigatórias
**Dado** que o projeto Android foi configurado
**Quando** se executa `grep -c "uses-permission" android/app/src/main/AndroidManifest.xml`
**Então** o resultado é ≥ 5 (BIOMETRIC, CAMERA, LOCATION, INTERNET, NETWORK_STATE)

### AC-005 — Workflow CI Android existe e é válido
**Dado** que o arquivo `.github/workflows/build-android.yml` existe
**Quando** se executa `cat .github/workflows/build-android.yml | grep "runs-on" | head -1`
**Então** o runner é `ubuntu-latest`

### AC-006 — Build release gera `.aab` assinado (com keystore configurada)
**Dado** que os secrets `KEYSTORE_BASE64`, `KEY_ALIAS`, `KEY_PASSWORD`, `STORE_PASSWORD` estão configurados no GitHub Actions
**Quando** o workflow `build-android.yml` é acionado manualmente
**Então** o job termina com exit 0 e um artefato `.aab` assinado é disponibilizado nos artifacts do GitHub Actions

_Nota: AC-006 não depende da conta Google Play Console — só da keystore. AC de upload ao Internal track requer conta Google, documentado como bloqueio externo._

## Evidências necessárias para aprovação

- `android/app/build/outputs/apk/debug/app-debug.apk` gerado (AC-001)
- Screenshot do emulador Android mostrando a tela de login
- `android/app/src/main/res/mipmap-xxxhdpi/ic_launcher.png` existente
- `.github/workflows/build-android.yml` com runner Ubuntu declarado
- `android/app/src/main/AndroidManifest.xml` com permissões declaradas
- Keystore **não** commitada no repositório (`.gitignore` inclui `*.keystore` e `*.jks`)

## Riscos

- **Keystore perdida = impossível atualizar app no Play Store** — mitigação: keystore armazenada em secret manager (1Password ou AWS Secrets) pelo PM, nunca só no CI
- **targetSdkVersion deve ser atualizado anualmente** — mitigação: monitorar Google Play policy; atualizar antes do prazo (geralmente agosto de cada ano)
- **ProGuard pode ofuscar e quebrar plugins Capacitor** — mitigação: regras ProGuard para Capacitor já documentadas pela Ionic; incluir no `proguard-rules.pro` antes do build release
- **Conta Google Play Console não provisionada** — mitigação: AC de upload é separado dos ACs de build; build local e CI passam sem a conta; documentar bloqueio

## Estimativa

14–20 horas de esforço (1 pessoa): configuração Gradle + Capacitor 5h, ícones adaptativos 3h, CI + Fastlane 6h, ProGuard + testes no emulador 4h. Upload ao Internal track adiciona 2h se conta Google estiver disponível.
