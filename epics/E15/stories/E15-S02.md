# E15-S02 — Scaffold React + TypeScript + Ionic + Capacitor + Vite

## Objetivo

Criar o esqueleto compilável do novo frontend Kalibrium nas versões validadas pelo spike INF-007 (E15-S01). O scaffold deve produzir um app que abre no navegador, em um emulador Android e em um simulador iOS — sem funcionalidade de negócio ainda, mas com toda a estrutura de rotas, layout adaptativo, configuração do Capacitor e pipeline de build funcionando. Este é o ponto de partida para todas as demais stories de E15.

Após esta story, qualquer desenvolvedor consegue clonar o repositório, rodar `npm install && npm run dev` e ver o app rodando — e rodar `npm run build:mobile` para gerar os artefatos Capacitor prontos para iOS e Android.

## Escopo

- Projeto React 18 + TypeScript 5 + Ionic 8 + Capacitor 6 + Vite 5, com versões exatas do `docs/frontend/stack-versions.md` (E15-S01)
- Estrutura de diretórios declarada: `src/`, `src/pages/`, `src/components/`, `src/hooks/`, `src/db/`, `src/auth/`, `src/wipe/`, `src/observability/`
- Roteamento com TanStack Router (ou Ionic Router conforme decisão do spike): rotas `/login`, `/home`, `/admin/devices`
- Layout adaptativo: componentes Ionic com breakpoints mobile-first + responsivo para tablet/desktop
- Configuração do Capacitor (`capacitor.config.ts`): `appId`, `appName`, `webDir`, plataformas iOS e Android declaradas
- Projeto Xcode gerado (`ios/`) via `npx cap add ios` — compilável localmente (exige macOS)
- Projeto Gradle gerado (`android/`) via `npx cap add android`
- Scripts npm: `dev` (Vite dev server), `build` (web), `build:ios` (Vite + cap sync + xcodebuild ad-hoc), `build:android` (Vite + cap sync + gradlew assembleRelease), `test` (Vitest), `test:e2e` (Playwright)
- Arquivo `docs/frontend/README.md` com instrução de setup e scripts
- Descarte formal dos arquivos Livewire/Blade legados: remover `resources/views/` e `resources/js/` (exceto arquivos necessários ao backend Laravel API-only)
- Configuração ESLint + Prettier para TypeScript/React

## Fora de escopo

- Service Worker e PWA manifest — E15-S03
- Build de distribuição no CI — E15-S04 e E15-S05

## Critérios de aceite

### AC-001 — App abre no navegador com rota `/login`
**Dado** que o desenvolvedor executa `npm install && npm run dev`
**Quando** acessa `http://localhost:5173/login` no Chrome
**Então** a página renderiza (componente Ionic, sem erro no console) com exit 0 no processo Vite

### AC-002 — Build web produz artefatos sem erros
**Dado** que o desenvolvedor executa `npm run build`
**Quando** o comando termina
**Então** existe o diretório `dist/` com `index.html` e assets, e o processo termina com exit 0

### AC-003 — Projeto iOS gerado e sincronizável
**Dado** que o desenvolvedor executa `npx cap add ios && npx cap sync ios` (em macOS)
**Quando** o comando termina
**Então** o diretório `ios/` existe com estrutura Xcode válida (`.xcworkspace` presente) e exit 0

### AC-004 — Projeto Android gerado e sincronizável
**Dado** que o desenvolvedor executa `npx cap add android && npx cap sync android`
**Quando** o comando termina
**Então** o diretório `android/` existe com estrutura Gradle válida (`build.gradle` presente) e exit 0

### AC-005 — Estrutura de diretórios conforme declarado
**Dado** que o scaffold foi criado
**Quando** se executa `ls src/`
**Então** os diretórios `pages/`, `components/`, `hooks/`, `db/`, `auth/`, `wipe/`, `observability/` existem (mesmo que vazios com `.gitkeep`)

### AC-006 — Layout adaptativo: mobile e desktop renderizam sem overflow
**Dado** que o app está rodando no dev server
**Quando** se abre o Chrome DevTools em viewport 375px (mobile) e 1280px (desktop) na rota `/home`
**Então** nenhum elemento ultrapassa o viewport (sem scroll horizontal) em ambas as resoluções — verificável via Playwright screenshot `tests/e2e/layout.spec.ts`

### AC-007 — ESLint e Prettier passam sem erros
**Dado** que o scaffold foi criado
**Quando** se executa `npm run lint`
**Então** o comando termina com exit 0 e zero erros reportados

### AC-008 — Frontend legado removido
**Dado** que o scaffold foi criado
**Quando** se executa `find resources/views -name "*.blade.php" 2>/dev/null | wc -l`
**Então** o resultado é `0` — ou o diretório `resources/views/` não existe; arquivos Blade legados não estão mais no repositório

## Evidências necessárias para aprovação

- `package.json` com versões exatas do `docs/frontend/stack-versions.md`
- `capacitor.config.ts` existente com `appId` e `appName` preenchidos
- Diretórios `ios/` e `android/` presentes e com estrutura válida
- `docs/frontend/README.md` com instrução de setup (mínimo: `npm install`, `npm run dev`, `npm run build`)
- Resultado de `npm run lint` com exit 0
- Resultado de `npm run test` com exit 0 (mesmo sem testes — zero testes passa)
- Screenshot Playwright do layout mobile e desktop

## Riscos

- **Conflito de peer dependencies entre Ionic 8 e React 18** — mitigação: versões validadas no spike E15-S01; se conflito emergir, ajustar para versão compatível documentada no spike
- **`npx cap add ios` requer macOS** — mitigação: CI terá runner macOS para iOS; desenvolvedor Windows/Linux usa só Android + web; documentar no README
- **Remoção do frontend legado pode quebrar rotas do backend** — mitigação: verificar `routes/web.php` antes de remover; backend deve servir só API (`routes/api.php`); qualquer rota web apontando para Blade deve ser removida ou redirecionada
