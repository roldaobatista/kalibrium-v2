# E15-S04 — Wrapper Capacitor iOS: projeto Xcode + build ad-hoc TestFlight

## Objetivo

Configurar o projeto Xcode gerado pelo Capacitor (E15-S02) para produzir um `.ipa` distribuível via TestFlight (ad-hoc / internal testing), com o CI (GitHub Actions runner macOS) realizando o build automaticamente a cada push na branch de release. Após esta story, qualquer pessoa com acesso ao TestFlight consegue instalar o app Kalibrium em um iPhone real e abrir a tela de login.

Esta story depende de **conta Apple Developer ativa** (pré-requisito externo do PM). Sem a conta, os ACs de build real e upload para TestFlight ficam pendentes — mas a configuração do projeto Xcode e do runner macOS no CI pode avançar.

## Escopo

- Configuração do `ios/App/App.xcworkspace` para build de produção: Bundle ID, versão, build number
- Provisioning profile ad-hoc e certificado de distribuição configurados via Fastlane Match (secrets no GitHub Actions)
- `Podfile` com dependências Capacitor — `pod install` executado sem erros
- Workflow GitHub Actions `.github/workflows/build-ios.yml`: trigger manual + trigger em tag `v*`
  - Runner: `macos-latest` (ou self-hosted macOS)
  - Steps: checkout → setup Node → npm install → npm run build → cap sync ios → pod install → fastlane build_adhoc → upload TestFlight
- Arquivo `fastlane/Fastfile` com lane `build_adhoc`
- Ícones iOS (AppIcon.appiconset) gerados conforme especificações Apple (20px a 1024px)
- Splash screen (`LaunchScreen.storyboard`) com logo Kalibrium
- Permissões declaradas no `Info.plist`: `NSFaceIDUsageDescription` (biometria — E15-S09), `NSCameraUsageDescription` (foto de evidência — futuro), `NSLocationWhenInUseUsageDescription` (GPS — futuro)
- Teste de smoke: app abre no simulador iOS e renderiza a rota `/login` sem crash

## Fora de escopo

- Biometria nativa (Face ID) — E15-S09
- SQLite nativo — E15-S06
- Autenticação funcional — E15-S07
- Submissão à App Store pública — ADR-0019 (decisão pós-E15)
- Contas Apple Developer — responsabilidade do PM

## Critérios de aceite

### AC-001 — `pod install` executa sem erros
**Dado** que o desenvolvedor está em macOS com CocoaPods instalado
**Quando** executa `cd ios/App && pod install`
**Então** o comando termina com exit 0 e `Pods/` é criado corretamente

### AC-002 — Build do simulador passa sem erro
**Dado** que o desenvolvedor está em macOS com Xcode instalado
**Quando** executa `npx cap run ios --target "iPhone 15 Simulator"`
**Então** o app abre no simulador iOS e a rota `/login` renderiza sem crash — verificado visualmente ou via XCUITest smoke test

### AC-003 — Ícones iOS presentes em todos os tamanhos obrigatórios
**Dado** que o projeto Xcode foi configurado
**Quando** se inspeciona `ios/App/App/Assets.xcassets/AppIcon.appiconset/Contents.json`
**Então** todos os tamanhos obrigatórios pela Apple estão declarados (20, 29, 40, 60, 76, 83.5, 1024 — em @1x, @2x, @3x conforme aplicável) e os arquivos PNG correspondentes existem

### AC-004 — Info.plist contém permissões declaradas
**Dado** que o projeto Xcode foi configurado
**Quando** se executa `grep -c "UsageDescription" ios/App/App/Info.plist`
**Então** o resultado é ≥ 3 (FaceID, Camera, Location declarados)

### AC-005 — Workflow CI iOS existe e é válido
**Dado** que o arquivo `.github/workflows/build-ios.yml` existe
**Quando** se executa `cat .github/workflows/build-ios.yml | grep "runs-on" | head -1`
**Então** o runner é `macos-latest` ou um runner macOS self-hosted

### AC-006 — Build ad-hoc gera `.ipa` (com conta Apple Developer provisionada)
**Dado** que os secrets `MATCH_PASSWORD`, `APPLE_ID`, `APP_STORE_CONNECT_API_KEY` estão configurados no GitHub Actions
**Quando** o workflow `build-ios.yml` é acionado manualmente
**Então** o job termina com exit 0 e um artefato `.ipa` é disponibilizado nos artifacts do GitHub Actions

_Nota: AC-006 fica pendente até PM provisionar conta Apple Developer. Os demais ACs (001-005) podem ser verificados sem a conta._

## Evidências necessárias para aprovação

- `ios/App/App.xcworkspace` existente com `Pods/` populado
- Screenshot do simulador iOS mostrando a tela de login
- `ios/App/App/Assets.xcassets/AppIcon.appiconset/Contents.json` com todos os tamanhos
- `.github/workflows/build-ios.yml` com runner macOS declarado
- `fastlane/Fastfile` com lane `build_adhoc`
- AC-006 marcado como pendente de conta Apple se não provisionada — documentado em `docs/frontend/README.md`

## Riscos

- **Conta Apple Developer não provisionada pelo PM** — mitigação: AC-006 é condicional; os demais ACs não dependem da conta; documentar bloqueio explicitamente
- **Runner macOS no GitHub Actions tem custo maior** — mitigação: trigger manual apenas (não em todo push); PM ciente do custo em ADR-0019
- **Xcode Command Line Tools version mismatch em CI** — mitigação: fixar versão do Xcode no workflow via `xcode-select`
- **CocoaPods deprecation em favor do Swift Package Manager** — mitigação: usar Capacitor com CocoaPods conforme recomendação oficial do Capacitor 6; monitorar notas de release para migração futura

## Estimativa

16–24 horas de esforço (1 pessoa): configuração Xcode + Capacitor 6h, ícones e splash 3h, Fastlane + CI 8h, testes no simulador 4h. AC-006 adiciona 3h se conta Apple estiver disponível.
