# E03-S07b — Endpoints consulta auditoria + RBAC + isolamento

## Objetivo

Expor os endpoints de consulta de audit log para os cadastros do E03, com paginação, RBAC restrito a admin e isolamento de tenant. Inclui tratamento de usuário removido e o endpoint de auditoria de instrumentos (SA-011).

## Escopo

- Endpoint `GET /auditoria/clientes/{id}` — lista eventos de audit do cliente com paginação (20 por página), ordenados por `created_at DESC`
- Endpoint `GET /auditoria/padroes/{id}` — idem para padrões de referência
- Endpoint `GET /auditoria/procedimentos/{id}` — idem para procedimentos de calibração
- Endpoint `GET /auditoria/instrumentos/{id}` — idem para instrumentos
- Os endpoints `/auditoria/contatos/{id}` estão fora do escopo desta story (declarado abaixo)
- Resposta de cada entrada inclui: `evento`, `user_id`, `user_nome`, `campos_alterados` (array com `campo`, `valor_anterior`, `valor_novo`), `criado_em`
- Tratamento de usuário removido: `user_nome = "(usuário removido)"` quando `user_id` não corresponde a usuário ativo — sem erro 500
- Policy RBAC: somente `admin` acessa endpoints de auditoria
- Isolamento: retorna 404 para entidade de outro tenant

## Fora de escopo

- Endpoint `/auditoria/contatos/{id}` — a implementação do audit de contatos via laravel-auditing é coberta por E03-S07a; a exposição via endpoint de consulta fica para pós-MVP (volume de dados pessoais exige análise de LGPD separada)
- Instalação do pacote e traits — E03-S07a
- Exportação em CSV — pós-MVP
- Audit de OS e calibrações — E05

## Criterios de aceite

### AC-005 — Audit registra criação e atualização de padrão de referência
**Dado** que o audit log está ativo
**Quando** admin cria um padrão e depois edita `certificado_validade`
**Então** existem 2 registros de audit para o padrão com os respectivos eventos

### AC-006 — Audit registra publicação de procedimento
**Dado** que "PG-DIM-001 v1.0" está em rascunho
**Quando** admin publica o procedimento (altera `status` de `rascunho` para `vigente`)
**Então** um registro de audit é criado com `old_values.status = rascunho` e `new_values.status = vigente`

### AC-007 — Endpoint de auditoria de cliente retorna histórico paginado
**Dado** que o cliente `C001` tem 25 registros de audit
**Quando** admin faz GET em `/auditoria/clientes/C001`
**Então** retornam 20 registros ordenados por `created_at DESC` com metadados de paginação

### AC-008 — Isolamento: admin só vê audit de entidades do seu tenant
**Dado** que sou admin do tenant A
**Quando** faço GET em `/auditoria/clientes/{id}` onde o cliente pertence ao tenant B
**Então** recebo HTTP 404

### AC-009 — RBAC: somente admin acessa endpoints de auditoria
**Dado** que sou usuário com role `atendente` ou `tecnico`
**Quando** faço GET em qualquer endpoint `/auditoria/*`
**Então** recebo HTTP 403

### AC-011 — Audit de instrumento registra criação e edição + endpoint acessível
**Dado** que o audit log está ativo nos models `Instrumento`
**Quando** atendente cria um instrumento e depois edita `numero_serie`, e admin faz GET em `/auditoria/instrumentos/{id}`
**Então** a resposta contém 2 registros de audit: `created` e `updated` com campo `numero_serie`; HTTP 200

### AC-012 — Usuário removido não quebra o audit log
**Dado** que existe um registro de audit cujo `user_id` pertence a um usuário que foi removido do tenant
**Quando** admin acessa o audit log
**Então** o registro é exibido com `user_id` preservado e `user_nome = "(usuário removido)"` — sem erro 500

## Testes obrigatórios

- Feature: GET /auditoria/clientes/{id} retorna 20 registros paginados para admin
- Feature: GET /auditoria/instrumentos/{id} retorna audit de instrumento criado e editado
- Feature: atendente/técnico recebe 403 em todos os endpoints de auditoria
- Feature: entity de outro tenant retorna 404
- Feature: user_id de usuário removido exibe "(usuário removido)" sem 500

## Riscos

- Usuário removido sem soft-delete pode causar FK violation — mitigação: `laravel-auditing` armazena `user_id` como integer sem FK constraint por padrão; tratamento via `leftJoin` ou `find()` com fallback
