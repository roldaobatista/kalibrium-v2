# E03-S03b — Listagens + filtros + paginação + isolamento de instrumento

## Objetivo

Expor as listagens de instrumentos (por cliente e global do tenant) com filtros, paginação e isolamento entre tenants. Após esta story, a recepção consegue localizar um instrumento pelo número de série independentemente de qual cliente o trouxe.

## Escopo

- Listagem de instrumentos por cliente (paginada: 20 por página) com filtro por modelo/número de série/domínio
- Listagem global de instrumentos do tenant (para recepção: "qual cliente trouxe este número de série?") com filtro e paginação
- Soft-delete via `ativo` (instrumento desativado não aparece na listagem padrão)
- Rota de exibição por ID retorna 404 para instrumento de outro tenant
- Campo `historico_calibracoes: []` incluso na resposta de exibição de instrumento

## Fora de escopo

- Model e validações de domínio — E03-S03a
- RBAC — incluído nesta story (abaixo)
- Execução de calibração — E05
- Audit log — E03-S07a

## Criterios de aceite

### AC-008 — Listagem de instrumentos por cliente com paginação
**Dado** que o cliente `C001` tem 25 instrumentos no tenant A
**Quando** listo instrumentos do cliente (página 1)
**Então** retorno tem 20 registros, todos com `cliente_id = C001`, e metadado de paginação indica página 2 disponível

### AC-009 — Filtro por domínio metrológico na listagem global
**Dado** que o tenant A tem instrumentos de 4 domínios diferentes
**Quando** filtro a listagem global por domínio `temperatura`
**Então** retornam apenas instrumentos com `dominio_metrologico = temperatura`

### AC-010 — Isolamento cross-tenant na listagem global
**Dado** que sou usuário do tenant A
**Quando** acesso a listagem global de instrumentos do tenant
**Então** nenhum instrumento do tenant B aparece (scope global ativo)

### AC-011 — Acesso a instrumento de outro tenant retorna 404
**Dado** que sou usuário do tenant A
**Quando** faço GET em `/instrumentos/{id}` onde o ID pertence a instrumento do tenant B
**Então** recebo HTTP 404

### AC-012 — Campo historico_calibracoes retorna lista vazia
**Dado** que existe instrumento `I001` recém-cadastrado
**Quando** faço GET em `/instrumentos/I001`
**Então** a resposta inclui `"historico_calibracoes": []` (placeholder para E05)

### AC-013 — RBAC: técnico não pode criar instrumento
**Dado** que sou usuário com role `tecnico`
**Quando** faço POST para criação de instrumento
**Então** recebo HTTP 403

### AC-014 — RBAC: técnico pode listar e ver instrumento
**Dado** que sou usuário com role `tecnico`
**Quando** faço GET na listagem de instrumentos de um cliente do meu tenant
**Então** recebo HTTP 200 com os dados

### AC-015 — Desativação de instrumento
**Dado** que existe instrumento ativo `I001`
**Quando** atendente desativa o instrumento
**Então** `ativo = false` é persistido e o instrumento não aparece na listagem padrão
