# E03-S05b — Alertas UI + endpoints consulta vencimento + scheduler

## Objetivo

Expor os alertas de vencimento na interface (badge de contagem no menu), endpoints de consulta de padrões vencidos e próximos do vencimento, ação de marcar alerta como visto e confirmação do agendamento do scheduler. Fecha o ciclo de alertas de vencimento visível para admin e técnico.

## Escopo

- Badge de contagem de alertas pendentes (`visto_em IS NULL`) no menu principal para roles `admin` e `tecnico`
- Listagem de alertas pendentes na interface
- Ação de marcar alerta como visto: PATCH `/alertas/{id}/visto` preenche `visto_em = now()` e `visto_por_user_id`
- Endpoint `GET /padroes/vencidos` lista padrões com `certificado_validade < hoje` e `ativo = true` do tenant
- Endpoint `GET /padroes/proximos-vencimento` lista padrões com `certificado_validade BETWEEN hoje AND hoje+30_dias` e `ativo = true` do tenant
- Policy RBAC: alertas visíveis para `admin` e `tecnico`; `atendente` não vê alertas de padrão
- Isolamento: alertas filtrados por `tenant_id` do usuário autenticado
- Confirmação de agendamento: job `VerificarVencimentoPadroes` agendado com `->dailyAt('06:00')->timezone('America/Sao_Paulo')` verificável via `artisan schedule:list`

## Fora de escopo

- Job de detecção e method estaVigente() — E03-S05a
- Envio de e-mail de alerta — E06
- Integração com OS para bloqueio efetivo — E05

## Criterios de aceite

### AC-005 — Alerta visível na interface para admin
**Dado** que existem 2 alertas não vistos no tenant A
**Quando** o admin do tenant A acessa qualquer página da aplicação
**Então** um badge com a contagem "2" é exibido no menu principal

### AC-006 — Alerta não visível para atendente
**Dado** que existem alertas de padrão vencido no tenant A
**Quando** um usuário com role `atendente` acessa a aplicação
**Então** o badge de alerta de padrão NÃO aparece no menu

### AC-007 — Marcar alerta como visto
**Dado** que existe alerta não visto `A001` no tenant A
**Quando** admin faz PATCH em `/alertas/A001/visto`
**Então** `visto_em = now()` e `visto_por_user_id` do admin são persistidos; o alerta não aparece mais na contagem de pendentes

### AC-008 — Isolamento de alertas entre tenants
**Dado** que sou admin do tenant A
**Quando** faço GET em `/alertas`
**Então** retornam apenas alertas com `tenant_id = tenant_A`; alertas do tenant B não aparecem

### AC-011 — Endpoint de padrões vencidos retorna apenas vencidos do tenant
**Dado** que sou admin do tenant A, com 2 padrões vencidos e 3 vigentes
**Quando** faço GET em `/padroes/vencidos`
**Então** retornam exatamente 2 registros, todos do tenant A

### AC-012 — Scheduler configurado para 06:00 BRT
**Dado** que o Scheduler do Laravel está ativo
**Quando** executo `artisan schedule:list`
**Então** o job `VerificarVencimentoPadroes` aparece agendado com frequência diária às 06:00 no fuso America/Sao_Paulo
